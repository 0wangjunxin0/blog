Commit 规范

推荐使用 Conventional Commits 规范：

类型 说明
feat 新功能
fix 修复 bug
docs 文档变更
style 代码格式（不影响逻辑）
refactor 代码重构
test 添加测试
chore 构建/工具变更

这些命令覆盖了 80% 的日常使用场景。遇到复杂情况，git --help 永远是你的朋友。
